## Claim Boundary

Allowed:

- Memory Doctor is a local Telegram-callable diagnostic prototype.
- It can distinguish active profile state from historical topic traces for the covered cases.
- It can flag and verify multi-delete integrity for recent Builder events.
- It can map the covered local movement path across gateway trace, Builder events, memory reads/writes, lifecycle/policy gates, context capsule source ledger, and watchtower dashboard state.
- It can identify covered lineage gaps when item-level evidence exists.
- It can verify that a fresh repaired provider capsule includes recent same-session Telegram conversation for the covered path.
- It can report weighted diagnostic-sense coverage and recommend proactive next probes when SDK, wiki, movement, gateway, active-profile, or capsule visibility is thin.
- It can preserve compact brain coverage/gap snapshots for later trend analysis without promoting them to memory truth.
- It can inspect LLM wiki packet metadata and LLM wiki health as supporting doctrine surfaces without letting wiki override current-state memory.

Forbidden:

- Memory Doctor can automatically repair memory.
- Memory Doctor is network absorbable.
- Historical trace mentions are safe to purge by default.
- Prototype evidence proves broad memory architecture quality.
- Memory Doctor has perfect visibility into unlogged or external systems.
- Movement-map gaps are repair actions by themselves.
- Treating a green incident-specific doctor result as proof that every dashboard aggregate is healthy.
- Brain coverage score proves the memory system is fully understood or globally healthy.
- Brain snapshot events can rewrite memory or trigger automatic repair without a separate authority gate.
- LLM wiki health proves mutable user facts or active runtime state.
