# Startup YC Privacy, Rollback, And Publication Review

Current publication boundary: workspace-only.

The Startup YC fixture can be used as a local reference and a bounded operator
capability example. It cannot become `network_absorbable` until the following
review is complete.

## Privacy Review

- Founder prompts, customer names, churn notes, revenue details, and investor
  context stay in the local or private lane by default.
- Public examples must be synthetic, anonymized, or explicitly approved.
- Spark must not ask an operator to paste secrets, tokens, or private customer
  records into creator-system docs.

## Rollback Review

- Deprecate the packet if multi-seed validation produces any negative track
  row for a `network_absorbable` claim.
- Deprecate the packet if held-out founder cases accept vanity metrics as PMF
  or recommend hiring/fundraising before demand evidence.
- Deprecate the packet if privacy review finds private founder/customer details
  in a shareable artifact.

## Publication Approval

Required before public or network sharing:

- named human/operator reviewer
- dated approval
- scope of approved examples
- approved evidence tier
- explicit statement that `network_absorbable` is still blocked unless every
  promotion gate in `validation_plan.json` passes
